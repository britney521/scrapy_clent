
# 分布式爬虫自动化管理系统开发文档

## 1. 项目概述
本项目是一个基于“服务端分发-客户端执行”架构的分布式网页自动化与数据采集系统。
系统分为管理后台（服务端）和执行节点（客户端）。客户端通过图形界面展示分配的任务，通过点击按钮驱动自动化引擎执行抓取，并将数据回传至服务端，同时在本地数据库留存状态和数据。

## 2. 核心技术栈选型
* **服务端 / 后台管理:** Python 3.10+、Django 5.x、Django REST Framework (DRF)
* **客户端 GUI:** PySide6 (基于 Qt for Python)
* **网页自动化引擎:** DrissionPage (无头/可视化浏览器控制)
* **客户端本地存储:** SQLite3

---

## 3. 核心业务流程
1. **任务获取:** 客户端启动后，通过 API 请求服务端获取分配给该账号的任务列表（包含任务 ID、目标 URL 等），并存入本地 SQLite。
2. **UI 渲染:** 客户端使用 PySide6 的 `QTableView` 展示任务列表，每行末尾生成操作按钮（如“运行爬虫”）。
3. **自动化执行:** 用户点击“运行爬虫”按钮，GUI 触发独立的工作线程（`QThread`）。线程内实例化 DrissionPage 执行数据采集。
4. **状态与数据管理:** 
   - 爬虫启动、成功、报错等状态需实时写入本地 SQLite，并通过信号（Signal）更新到 UI 表格。
   - 采集到的业务数据先持久化到 SQLite，随后通过 API 提交到 Django 服务端。提交成功后标记同步状态。

## 4. 数据库设计 (Database Schema)

### 4.1 Django 服务端模型 (Models)
需要创建以下数据模型来管理任务和回收的数据：

```python
from django.db import models
from django.contrib.auth.models import User

class ClientTask(models.Model):
    """分配给客户端的任务表"""
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    task_name = models.CharField(max_length=255)
    target_url = models.URLField()
    status = models.CharField(max_length=50, default='PENDING') # PENDING, RUNNING, SUCCESS, FAILED
    created_at = models.DateTimeField(auto_now_add=True)

class ScrapedData(models.Model):
    """存储客户端回传的业务数据"""
    task = models.ForeignKey(ClientTask, on_delete=models.CASCADE)
    result_data = models.JSONField(verbose_name="采集到的JSON数据")
    uploaded_at = models.DateTimeField(auto_now_add=True)

class TaskLog(models.Model):
    """记录客户端运行日志"""
    task = models.ForeignKey(ClientTask, on_delete=models.CASCADE)
    message = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

```

### 4.2 客户端 SQLite 本地表结构

```sql
-- 本地任务表
CREATE TABLE local_tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER UNIQUE,
    target_url TEXT,
    status TEXT DEFAULT 'PENDING',
    last_error TEXT
);

-- 本地数据缓存表
CREATE TABLE local_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER,
    data_payload TEXT,
    is_synced INTEGER DEFAULT 0 -- 0: 未同步, 1: 已同步
);

```

## 5. API 接口契约

服务端需提供以下三个 RESTful API：

1. **获取任务接口 (`GET /api/tasks/`)**
* 作用：拉取当前用户的任务列表。


2. **上报状态接口 (`POST /api/tasks/{id}/status/`)**
* 作用：回传任务的执行状态和错误日志。


3. **提交数据接口 (`POST /api/tasks/{id}/data/`)**
* 作用：将抓取到的 JSON 数组上报并存入 `ScrapedData` 表。

## 6. 客户端核心实现规范 (PySide6 + DrissionPage)

为了保证 UI 不卡顿，**必须使用多线程**运行爬虫逻辑：

```python
from PySide6.QtCore import QThread, Signal
from DrissionPage import ChromiumPage, ChromiumOptions

class SpiderWorker(QThread):
    """执行 DrissionPage 爬虫的独立后台线程"""
    status_changed = Signal(int, str, str) # task_id, status, error_msg
    data_fetched = Signal(int, list)       # task_id, result_data_list

    def __init__(self, task_id, target_url):
        super().__init__()
        self.task_id = task_id
        self.target_url = target_url

    def run(self):
        self.status_changed.emit(self.task_id, "RUNNING", "")
        try:
            # 1. 启动 DrissionPage
            co = ChromiumOptions()
            page = ChromiumPage(co)
            
            # 2. 执行自动化逻辑
            page.get(self.target_url)
            # TODO: 编写页面解析逻辑
            mock_data = [{"title": "示例数据", "url": self.target_url}]
            page.quit()
            
            # 3. 发送成功数据和状态
            self.data_fetched.emit(self.task_id, mock_data)
            self.status_changed.emit(self.task_id, "SUCCESS", "")
        except Exception as e:
            self.status_changed.emit(self.task_id, "FAILED", str(e))

```

在 `MainWindow` 视图中，使用 `QTableView` 配合 `QAbstractTableModel` 读取 SQLite 数据，通过 `QStyledItemDelegate` 在表格列渲染“启动”按钮，并与 `SpiderWorker` 进行信号绑定。
